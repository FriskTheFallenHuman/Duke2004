virtual UBOOL FQueuedThreadMac::Kill ( UBOOL  bShouldWait = FALSE,
UBOOL  bShouldDeleteSelf = FALSE 
)
virtual

Tells the thread to exit. If the caller needs to know when the thread has exited, it should use the bShouldWait value and tell it how long to wait before deciding that it is deadlocked and needs to be destroyed. NOTE: having a thread forcibly destroyed can cause leaks in TLS, etc.

Parameters
bShouldWaitIf true, the call will wait for the thread to exit
bShouldDeleteSelfWhether to delete ourselves upon completion
Returns
True if the thread exited gracefull, false otherwise
