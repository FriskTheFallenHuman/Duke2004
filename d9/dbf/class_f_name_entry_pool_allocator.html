Detailed Description

Pooled allocator for FNameEntry structures. Doesn't have to worry about freeing memory as those never go away. It simply uses 64K chunks and allocates new ones as space runs out. This reduces allocation overhead significantly (only minor waste on 64k boundaries) and also greatly helps with fragmentation as 50-100k allocations turn into tens of allocations.
