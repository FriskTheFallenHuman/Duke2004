◆ LargestExpectedViewportWidth

UINT FD3D9DynamicRHI::LargestExpectedViewportWidth
private

The largest viewport size the application is expecting to need for the time being, or zero for no preference. This is used as a hint to the RHI to reduce redundant device resets when viewports are created or destroyed (typically in the editor.) These values can change at any point, but they're usually configured before a viewport is created or destroyed.
