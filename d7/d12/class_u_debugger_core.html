◆ CurrentState

FDebuggerState* UDebuggerCore::CurrentState
protected

The state the UDebugger is currently in. States determine how the UDebugger behaves; the most important states are: DSIdleState: UDebugger is attached and the game is running normally DSWaitForInput: Game is paused and UDebugger is currently in interactive mode, waiting for user input (i.e. step into, out of) DSStep*: Game is running normally but UDebugger will automatically pause when a certain class/line is reached.
