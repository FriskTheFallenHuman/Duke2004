Utility function to compress an animation. If the animation is currently associated with a codec, it will be used to compress the animation. Otherwise, the default codec will be used. If AllowAlternateCompressor is TRUE, an alternative compression codec will also be tested. If the alternative codec produces better compression and the accuracy of the compressed animation remains within tolerances, the alternative codec will be used. See GetAlternativeCompressionThreshold for information on the tolerance value used.

Parameters
AnimSeqThe anim sequence to compress.
SkelMeshThe skeletal mesh against which to compress the animation. Not needed by all compression schemes.
AllowAlternateCompressorTRUE if an alternative compressor is permitted.
bOutputIf FALSE don't generate output or compute memory savings.
Returns
None.

Utility function to compress an animation. If the animation is currently associated with a codec, it will be used to compress the animation. Otherwise, the default codec will be used. If AllowAlternateCompressor is TRUE, an alternative compression codec will also be tested. If the alternative codec produces better compression and the accuracy of the compressed animation remains within tolerances, the alternative codec will be used. See GetAlternativeCompressionThreshold for information on the tolerance value used.

Parameters
AnimSeqThe anim sequence to compress.
SkelMeshThe skeletal mesh against which to compress the animation. Not needed by all compression schemes.
MasterToleranceThe alternate error threshold (0.0 means don't try anything other than the current / default scheme)
bOutputIf FALSE don't generate output or compute memory savings.
bFirstRecompressUsingCurrentOrDefaultIf TRUE, then the animation will be first recompressed with it's current compressor if non-NULL, or with the global default compressor (specified in the engine ini)
bForceBelowThresholdIf TRUE and the existing compression error is greater than MasterTolerance, then any compression technique (even one that increases the size) with a lower error will be used until it falls below the threshold
bRaiseMaxErrorToExistingIf TRUE and the existing compression error is greater than MasterTolerance, then MasterTolerance will be effectively raised to the existing error level
bTryFixedBitwiseCompressionIf TRUE, the uniform bitwise techniques will be tried
bTryPerTrackBitwiseCompressionIf TRUE, the per-track compressor techniques will be tried
bTryLinearKeyRemovalCompressionIf TRUE, the linear key removal techniques will be tried
bTryIntervalKeyRemovalIf TRUE, the resampling techniques will be tried
Returns
None.

FLOAT FAnimationUtils::GetAlternativeCompressionThreshold ( )
static

Determines the current setting for world-space error tolerance in the animation compressor. When requested, animation being compressed will also consider an alternative compression method if the end result of that method produces less error than the AlternativeCompressionThreshold. The default tolerance value is 0.0f (no alternatives allowed) but may be overriden using a field in the base engine INI file.

Returns
World-space error tolerance for considering an alternative compression method

Parameters

Returns
TRUE if the keys are uniformly spaced (or one of the trivial conditions is detected). FALSE if any key spacing is greater than 1e-4 off.
